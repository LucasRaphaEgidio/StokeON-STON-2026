O que é Integração Contínua?
R= A Integração Contínua (CI) é uma prática de desenvolvimento em que as alterações de código são mescladas frequentemente em um repositório central. A cada nova integração, 
processos automatizados de build e testes são acionados para validar as mudanças, ajudando a detectar e corrigir erros de forma rápida e contínua.

Qual é o papel das branches nesse processo?
R=As branches funcionam como ambientes de trabalho isolados e seguros. Elas permitem que os desenvolvedores criem novas funcionalidades,
experimentem ou corrijam falhas sem o risco de alterar, quebrar ou instabilizar a versão principal do código do projeto.

Por que o Pull Request é importante?
R=O Pull Request (PR) é essencial porque atua como um ponto de controle e colaboração. Ele avisa a equipe de que o trabalho em uma branch foi concluído, permitindo que outros desenvolvedores revisem o código,
discutam melhorias e garantam a qualidade das alterações antes que elas sejam definitivamente aprovadas e incorporadas à base principal.
