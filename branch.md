A Integração Contínua (CI) é uma prática de desenvolvimento ágil onde os membros de uma equipe integram seus códigos no repositório principal de forma frequente, permitindo a detecção rápida de erros de compatibilidade. Nesse fluxo, as branches exercem o papel de isolar o trabalho de cada funcionalidade ou correção de forma segura, evitando que modificações incompletas afetem diretamente o código estável da ramificação principal. O Pull Request é fundamental pois serve como uma etapa de revisão e validação, permitindo que a equipe inspecione o código, discuta melhorias e garanta a qualidade do software antes que a integração final seja consolidada.
